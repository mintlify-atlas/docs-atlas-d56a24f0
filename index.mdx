title: "web-vitals: Core Web Vitals for real user monitoring"
description: "Measure Core Web Vitals on real users with a tiny, modular JavaScript library that accurately matches Chrome's measurement methodology."
---

The `web-vitals` library makes it easy to measure all [Core Web Vitals](https://web.dev/articles/vitals) and key performance metrics on real users. At just ~2KB (brotli'd), it gives you accurate, actionable data that matches exactly what Chrome reports to tools like PageSpeed Insights, Search Console, and CrUX.

## Core Web Vitals

The three metrics that matter most for user experience — and that Google uses for Search ranking:

<CardGroup cols={3}>
  <Card title="CLS" icon="arrows-up-down-left-right" href="/api/on-cls">
    **Cumulative Layout Shift** — Measures visual stability. How much does page content unexpectedly shift?
  </Card>
  <Card title="INP" icon="computer-mouse" href="/api/on-inp">
    **Interaction to Next Paint** — Measures responsiveness. How quickly does the page respond to interactions?
  </Card>
  <Card title="LCP" icon="image" href="/api/on-lcp">
    **Largest Contentful Paint** — Measures loading performance. How fast does the main content appear?
  </Card>
</CardGroup>

## Additional Metrics

<CardGroup cols={2}>
  <Card title="FCP" icon="paintbrush" href="/api/on-fcp">
    **First Contentful Paint** — Time until the first content is rendered on screen.
  </Card>
  <Card title="TTFB" icon="server" href="/api/on-ttfb">
    **Time to First Byte** — Time from navigation start until the first byte of response is received.
  </Card>
</CardGroup>
