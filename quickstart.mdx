---
title: "Quickstart: install web-vitals and start measuring"
description: "Install web-vitals, measure Core Web Vitals on real users, and send the data to your analytics endpoint in a few lines of JavaScript."
---

This guide walks you through installing `web-vitals`, measuring the Core Web Vitals (CLS, INP, LCP) and the two additional metrics (FCP, TTFB), and reporting the results to an analytics endpoint using the `navigator.sendBeacon()` API.

<Steps>
  <Step title="Install">
    Install the `web-vitals` package from npm.

    <CodeGroup>

    ```bash npm
    npm install web-vitals
    ```

    ```bash yarn
    yarn add web-vitals
    ```

    ```bash pnpm
    pnpm add web-vitals
    ```

    </CodeGroup>
  </Step>

  <Step title="Measure Core Web Vitals">
    Import the three Core Web Vitals functions and pass a callback. Each callback receives a `Metric` object when the value is ready to report.

    ```js
    import {onCLS, onINP, onLCP} from 'web-vitals';

    onCLS(console.log);
    onINP(console.log);
    onLCP(console.log);
    ```

    Each logged object includes `name`, `value`, `rating` (`"good"`, `"needs-improvement"`, or `"poor"`), `delta`, and `id`.
  </Step>

  <Step title="Measure all metrics">
    Add `onFCP` and `onTTFB` to collect the full set of Web Vitals.

    ```js
    import {onCLS, onFCP, onINP, onLCP, onTTFB} from 'web-vitals';

    onCLS(console.log);
    onFCP(console.log);
    onINP(console.log);
    onLCP(console.log);
    onTTFB(console.log);
    ```
  </Step>

  <Step title="Send to an analytics endpoint">
    Replace `console.log` with a function that sends each metric to your backend. The example below uses [`navigator.sendBeacon()`](https://developer.mozilla.org/docs/Web/API/Navigator/sendBeacon), which is available across all modern browsers and supports sending data reliably as the page unloads.

    ```js
    import {onCLS, onINP, onLCP} from 'web-vitals';

    function sendToAnalytics(metric) {
      const body = JSON.stringify({
        name: metric.name,
        value: metric.value,
        id: metric.id,

        // Include additional data as needed...
      });

      // Use `navigator.sendBeacon()` to send the data, which supports
      // sending while the page is unloading.
      navigator.sendBeacon('/analytics', body);
    }

    onCLS(sendToAnalytics);
    onINP(sendToAnalytics);
    onLCP(sendToAnalytics);
    ```

    The `id` field uniquely identifies the metric instance for the current page load and can be used to deduplicate or aggregate values in your analytics backend.
  </Step>
</Steps>

<Note>
  Do not call any Web Vitals function (e.g. `onCLS()`, `onINP()`, `onLCP()`) more than once per page load. Each call registers a `PerformanceObserver` and event listeners that persist for the lifetime of the page. Calling them repeatedly may result in a memory leak.
</Note>

<Note>
  Some metrics may not report immediately or at all. INP is only reported after the user interacts with the page. CLS, FCP, and LCP are not reported if the page was loaded in the background. If you don't see values logged to the console, try reloading with **Preserve log** enabled in DevTools, or switch tabs and switch back to trigger a `visibilitychange` event.
</Note>
